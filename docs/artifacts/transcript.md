# `transcript.json`

## Purpose

`transcript.json` stores time-aligned transcript text for a media input.

The current implementation may write a stub transcript. Future real transcribers must write the same schema or introduce a new explicit schema version.

## Schema Version

Current schema version:

```text
transcript.v1
```

## Fields

- `schema_version`: transcript schema identifier. Current value is `transcript.v1`.
- `source`: source metadata for the transcript artifact.
- `language`: detected or declared language. Current stub value is `unknown`.
- `duration_seconds`: transcript/media duration when known. Current stub value is `null`.
- `segments`: ordered transcript segments.

## Source Object

Current source fields:

- `input_path`: path to the input media file used by the worker.
- `mode`: worker mode. Current values are `stub` or `real`.

Optional source fields for real transcription:

- `engine`: transcription engine, such as `faster-whisper`.
- `model_size`: model size used by the transcription engine, such as `tiny`.

Future source fields may identify worker implementation, model, parameters, or source artifact references.

## Segment Fields

Each segment includes:

- `id`: stable segment identifier within the transcript.
- `start`: start time in seconds.
- `end`: end time in seconds.
- `text`: transcript text.

Segments should be ordered by start time.

## Validation Expectations

The Go control plane validates real transcription output before marking a run completed.

Validation checks:

- `schema_version` is `transcript.v1`
- `source.mode` exists
- `segments` exists and is an array
- each segment has `id`, `start`, `end`, and `text`
- `start` and `end` are numbers
- `text` is a string
- `end >= start`

Segment timing is measured in seconds. Segment boundaries should not be negative in future stricter validators, but the current validator only requires numeric values and `end >= start`.

## Example

```json
{
  "schema_version": "transcript.v1",
  "source": {
    "input_path": "/Users/example/BYOMVIDEO/examples/fixtures/tiny.mp4",
    "mode": "stub"
  },
  "language": "unknown",
  "duration_seconds": null,
  "segments": [
    {
      "id": "seg_0001",
      "start": 0.0,
      "end": 2.0,
      "text": "Stub transcript generated by BYOM Video."
    }
  ]
}
```

Real transcription example:

```json
{
  "schema_version": "transcript.v1",
  "source": {
    "input_path": "/Users/example/BYOMVIDEO/examples/fixtures/tiny.mp4",
    "mode": "real",
    "engine": "faster-whisper",
    "model_size": "tiny"
  },
  "language": "en",
  "duration_seconds": 2.0,
  "segments": [
    {
      "id": "seg_0001",
      "start": 0.0,
      "end": 1.8,
      "text": "Example transcribed text."
    }
  ]
}
```

## Future Real Transcribers

Real transcription workers should preserve this contract:

- write `transcript.json` into the run directory
- use `schema_version: transcript.v1` when compatible
- preserve `source`, `language`, `duration_seconds`, and `segments`
- keep segment `id`, `start`, `end`, and `text`

If a future transcription format needs incompatible fields or semantics, it should use a new schema version.
